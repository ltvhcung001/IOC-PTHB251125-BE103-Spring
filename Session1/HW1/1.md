1. Client (Trình duyệt)
- Hành động: Người dùng nhập địa chỉ trang web hoặc nhấn vào menu "Sản phẩm".

- HTTP Method: GET

- URL dự kiến: /api/products (nếu là API) hoặc http://localhost:8080/products (nếu là trang web truyền thống).

- Cách thức: Trình duyệt đóng gói các thông tin cần thiết vào một HTTP Request và gửi qua môi trường mạng (Internet/LAN) tới Server.

2. Server (Ứng dụng Spring Boot)
- Khi nhận được yêu cầu từ Client, Spring Boot sẽ thực hiện các bước sau:
    
    - Tiếp nhận (Controller): Lớp Controller sẽ nhận Request dựa trên URL và Method tương ứng.
    
    - Xử lý logic (Service): Gọi đến lớp Service để xử lý các nghiệp vụ (ví dụ: lọc sản phẩm còn hàng, tính toán giảm giá nếu có).

    - Truy xuất dữ liệu (Repository): Gọi lớp Repository để lấy dữ liệu từ "Database".
Điều phối: Sau khi có dữ liệu, Server sẽ chuẩn bị dữ liệu đó để gửi trả về cho Client.
3. Database
- Vai trò: Là nơi lưu trữ tập trung dữ liệu về sản phẩm.

- Hoạt động: Khi Server yêu cầu, Database (RAM) sẽ trả về danh sách các đối tượng (Object) sản phẩm bao gồm các thông tin như: id, name, price, description,...

4. Response (Phản hồi từ Server)
- Sau khi xử lý xong, Server gửi trả về một HTTP Response cho Client:

- Định dạng: JSON. Nếu ứng dụng làm theo mô hình REST API, Server trả về chuỗi JSON để Client (React, Angular, Mobile app...) tự hiển thị.

- HTML: Nếu ứng dụng sử dụng Server-side rendering (như Thymeleaf), Server sẽ trả về một file HTML hoàn chỉnh chứa danh sách sản phẩm.

- Trạng thái (Status Code): 200 OK (Biểu thị yêu cầu đã được xử lý thành công).
